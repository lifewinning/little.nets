I was taught to estimate instruction timing by counting memory referrences.  You 
could one or two for fetching the instruction; you count one if it fetches a 
variable from memory; if it is deferred addressing, you count one more!

You cannot count, like old school programmers were taught.  However, when 
starting with empty CPU caches, count for me the number of memory referrences.

        MOV     RBX,U64 [RAX]

The instruction is probably held withing one memory cacheline.
There is a fetch of page table a level 3
there is a fetch from the 2Meg page table
there is a fetch from the 4K page table
there is a fetch of the instruction's physical address.

Now, the data pointed to by RAX must be fetched.
There is a fetch of page table a level 3
there is a fetch from the 2Meg page table
there is a fetch from the 4K page table
there is a fetch of the data physical address.

It takes 8 memory fetches for this instruction:

        MOV     RBX,U64 [RAX]
