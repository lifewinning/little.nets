                                    <Native>
Burn a CD/DVD from TempleOSCD.ISO and boot it to run natively.


                                    <VMware>
Use VMware to run the ISO in a virtual machine.  Aim the CD/DVD at 
TempleOSCD.ISO.  Downgrade to VMware Player 4.0 or VMware Workstation 8.0 if you 
want sound and you have an internal PC speaker.

Buy VMware Workstation for, like $250, and you can mount a drive to move files 
in and out.  Multicore works in VMware.  I use VMware Workstation in Ubuntu.  
Sound does not work.

You might use WinImage or other software to get files in and out if you do not 
purchase Workstation.

We need to petition VMware to support PC Speaker.


                                  <VirtualBox>
Use VirtualBox to run the ISO in a virtual machine by aiming the CD/DVD at 
TempleOSCD.ISO.


                                     <QEMU>
Use QEMU to run the ISO in a virtual machine.  QEMU does not support multicore.

Wiki on QEMU Images

You need to make a file for your guest hard disk image.  This command creates a 
file which is where you are going to install onto from the CDROM ISO file.  It 
will behave as the hard drive for the guest operating system. 
>qemu-img create -f qcow2 my_disk.img 3G

The first time, boot the CDROM ISO and install on a virtual machine.  (Type this 
command on one line.)
>qemu-system-x86_64 -hda my_disk.img -enable-kvm -m 512 -cpu host -soundhw pcspk 
-cdrom TempleOSCD.ISO -boot d

After installing, boot the hard disk.
>qemu-system-x86_64 -hda my_disk.img -enable-kvm -m 512 -cpu host -soundhw pcspk

This connects the image to /dev/nbd0.  It acts like any other block device.  It 
mounts it to mount point, /mnt/my_disk.
>sudo modprobe nbd max_part=16
>sudo qemu-nbd -c /dev/nbd0 my_disk.img
>sudo partprobe /dev/nbd0
>sudo mkdir /mnt/my_disk
>sudo mount /dev/nbd0p1 /mnt/my_disk
