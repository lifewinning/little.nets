CDC's have a 4x4 matrix for rotating, scaling, skewing and shifting in 3 
dimensions.  To make the graphics routines use the transform, you must set the 
DCF_TRANSFORMATION flag.

The matrix consists of integers that have been scaled 32 bits (GR_SCALE).  See 
::/Demo/Lectures/FixedPoint.CPP to learn why.

See GrRotXNew(), GrRotYNew(), GrRotZNew(), GrScaleNew(), GrRotXEqu(), GrRotYEqu(
), GrRotZEqu() and GrScaleEqu() to rotate about axes and scale.  Combine them 
with GrMulMat() and assign them to the CDC.dc with GrSetRotMat().  See 
::/Demo/Graphics/Box.CPP.

You can rotate single points using GrRot().

The 4th dimension allows a neat trick where you can place pos shifts 
(translations), into the matrix and GrMulMat them to combine rotation/shift 
operations.  Normally, you can't combine pos shift operations.  See 
GrSetTranslation() and ::/Demo/Graphics/Transform.CPP.

Finally, CDC's have an x, y and z which is an additional shift (translation).

The transformation is implemented as a callback on the CDC's transform() member. 
 The default transfrom() callback is GrTransform().  See 
::/Demo/Games/EagleDive.CPP or ::/Demo/Games/CastleFrankenstein.CPP to see how 
to change the transfrom() callback for foreshortening.
